01

Memory map architecture

CMIS uses a two-window address model over I²C. The lower 128 bytes (0–127) are always accessible without paging. Byte 127 is the page select register that switches the upper 128 bytes (128–255) to different page views.

03

Module state machine

Before any data path can be configured, the module must reach ModuleReady. The host drives this by controlling the LPMode hardware signal and writing to control registers. State is reported in Lower byte 3 bits [2:0].

04

Data path state machine

Each lane has its own DP state machine running inside the module after ModuleReady. State is reported in Lower byte 26, two bits per lane. Step through all five states below.

05

Application select procedure

The 5-step sequence to bring a lane from Deactivated to DataPathActivated. The DPDeinit pulse (assert then de-assert) is the trigger that starts module initialization.

06

VDM — Versatile Diagnostic Monitoring

VDM allows the module to report any combination of up to 32 observable parameters (Pre-FEC BER, SNR, CD, OSNR, bias, temperature …) chosen from a standard catalog of Observable IDs. The host reads the descriptor table to know which observables are present, then reads sample values from the corresponding bytes.

07

CDB — Command Data Buffer protocol

CDB is a mailbox protocol for extended management operations not covered by the basic memory map — firmware update, detailed diagnostics, loopback control, and vendor commands. The host writes a command block to Page 09h, triggers it via Lower byte 38, polls for completion, then reads the reply from Page 04h.

08

CDB command code reference

Standard CDB command codes defined by CMIS 5.2. Vendor-specific commands use codes in the 0x8000–0xFFFF range. The host writes the command code to Page 09h bytes 128–129 (MSB first).

0x0000QueryRead module capabilities — supported CDB commands, max LPL/EPL size, firmware info
0x0001Get FW infoReturns currently running and committed firmware version strings
0x0100Enter FW update modePuts module into firmware download mode; stops normal operation on selected image slot
0x0101FW download startInitiates firmware block transfer; returns target image slot and expected size
0x0103FW download completeSignals end of firmware block transfer; module validates CRC
0x0104FW runCommands module to boot from newly downloaded image (may trigger reset)
